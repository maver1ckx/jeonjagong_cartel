# jeonjagong_cartel Project Repository Guide

이 레포지토리는 프로젝트의 **코드, 문서, 회의록 및 각 팀별 작업 결과물**을 통합 관리합니다.
각 폴더는 기능 및 팀 단위로 구성되어 있으며, 관련 자료는 해당 폴더 내에 정리됩니다.

---
## 💬 커밋 메세지 규칙

커밋 메세지는 다음과 같은 규칙을 따라 작성합니다.

* feat : 기능 추가
* fix : 버그 수정
* refactor: 코드 구조 개선
* style: UI / 코드 스타일 변경
* docs: 문서 수정

#### EX)
feat: add tilt rotation
fix: pan rotation not working issue
refactor: improve pan rotation logic
style: update control UI
docs: add README

## 📂 폴더 구조

### 🔧 Embedded

임베디드 시스템 관련 코드 및 문서를 관리합니다.

* 펌웨어 소스 코드
* MCU 제어 로직
* 센서 및 액추에이터 제어
* 관련 기술 문서

---

### 🌐 Network

네트워크 및 통신 관련 작업을 관리합니다.

* 서버/클라이언트 통신 코드
* 프로토콜 정의
* 데이터 송수신 처리
* 네트워크 구조 문서

---

### 🖥 UI & API

사용자 인터페이스 및 API 관련 작업을 관리합니다.

* 프론트엔드 코드
* 백엔드 API 구현
* UI 설계 및 흐름
* API 명세 문서

---

### 🔌 Hardware

하드웨어 설계 및 제작 관련 자료를 관리합니다.

* 회로도 및 PCB 설계
* 부품 리스트 (BOM)
* 하드웨어 테스트 결과
* 제작 관련 문서

---

## 📝 공통 문서

### 📌 회의록 (Meetings)

* 회의 내용 기록
* 결정 사항 및 이슈 정리
* TODO 및 역할 분배

---

### 📄 기타 문서 (Docs)

* 프로젝트 기획서
* 요구사항 정의서
* 참고 자료 및 리서치 내용

---

## 📏 규칙

* 각 팀은 자신의 폴더에 관련 자료를 정리합니다.
* 문서 및 회의록은 최신 상태로 유지합니다.
* 파일명은 의미 있게 작성합니다. (예: `2026-04-07_meeting.md`)
* 중요한 변경 사항은 커밋 메시지에 명확히 작성합니다.

---

## 🎯 목적

이 레포지토리는
👉 **팀 간 협업을 효율적으로 만들고, 모든 작업 이력을 체계적으로 관리하는 것**을 목표로 합니다.
